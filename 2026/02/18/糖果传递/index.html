糖果传递

    • 这是一个经典的数学建模问题，通常被称为**“环形均分纸牌问题”**。

      这道题初看是一个复杂的分配问题，但通过数学推导，我们可以将其转化为一个中位数问题。下面我将为你详细拆解解题思路。

      一、 核心思路分析

      1. 目标分析

      我们要让每个人的糖果数最后都等于平均值 avg。
      假设第 $i$ 个小朋友初始有 $a_i$ 个糖果，总人数为 $n$。
      平均值 $avg = \frac{\sum a_i}{n}$。

      2. 建立变量

      由于每个人只能给左右两边传递糖果，我们定义 $x_i$：

      • $x_i$ 表示第 $i$ 个小朋友给第 $i+1$ 个小朋友的糖果数量。
      • 如果 $x_i > 0$，表示 $i$ 给了 $i+1$ 糖果；
      • 如果 $x_i < 0$，表示 $i+1$ 给了 $i$ 糖果（负负得正）；
      • 特别地，$x_n$ 表示第 $n$ 个小朋友给第 $1$ 个小朋友的数量（因为是圆圈）。

      3. 列出方程

      对于每个小朋友，他最后的糖果数应该等于 $avg$：

      • 第 1 个人：$a_1 - x_1 + x_n = avg$ $\Rightarrow$ $x_1 = x_n + a_1 - avg$
      • 第 2 个人：$a_2 - x_2 + x_1 = avg$ $\Rightarrow$ $x_2 = x_1 + a_2 - avg = x_n + (a_1 + a_2 - 2avg)$
      • 第 3 个人：$a_3 - x_3 + x_2 = avg$ $\Rightarrow$ $x_3 = x_2 + a_3 - avg = x_n + (a_1 + a_2 + a_3 - 3avg)$
      • …
      • 第 $i$ 个人：$x_i = x_n + (\sum_{j=1}^i a_j - i \cdot avg)$

      4. 简化公式

      令 $C_i = \sum_{j=1}^i a_j - i \cdot avg$。
      注意，$C_i$ 是可以提前算出来的常数（即前 $i$ 个小朋友初始糖果与目标总糖果的差值）。
      那么：
      $x_1 = x_n + C_1$
      $x_2 = x_n + C_2$
      …
      $x_{n-1} = x_n + C_{n-1}$
      $x_n = x_n + C_n$ （由于总数不变，$C_n$ 必然为 0）

      5. 转化代价函数

      题目要求的最小代价是传递糖果的总次数，即 $\sum_{i=1}^n |x_i|$。
      代入上面的简化公式：
      $$代价 = |x_1| + |x_2| + \dots + |x_n| = |x_n + C_1| + |x_n + C_2| + \dots + |x_n + C_{n-1}| + |x_n + 0|$$
      整理一下形式：
      $$代价 = |x_n - (-C_1)| + |x_n - (-C_2)| + \dots + |x_n - (-C_{n-1})| + |x_n - 0|$$

      关键点： 这个式子的物理含义是：在数轴上找一个点 $x_n$，使得它到点 ${0, -C_1, -C_2, \dots, -C_{n-1}}$ 的距离之和最小。

      根据经典数学结论：数轴上一个点到一组点距离之和最小，该点必然是这组点的中位数。

      二、 解题步骤总结

      1. 计算糖果平均值 avg。
      2. 计算数组 $C$ 的各个元素：$C_i = C_{i-1} + a_i - avg$。
      3. 对数组 $C$ 进行排序。
      4. 找到 $C$ 的中位数 $mid$。
      5. 累加计算每个 $|C_i - mid|$ 的绝对值，结果即为最小代价。

      三、 C++ 代码实现

      代码中使用了 long long 以防止数据溢出（$a_i$ 最大 20 亿，累加会非常大）。

      四、 关键点解析（新手必看）

      1. 为什么是中位数？
        想象你在一条路上放了几个车站，你想盖一间房子，让房子到所有车站的总路程最短。如果你把房子放在最左边，往右移动一点点，到左边车站的距离增加了，但到右边所有车站的距离都减少了。只要右边的车站比左边多，往右移就是赚的。直到左边和右边的车站一样多时，你就达到了最优位置。这就是中位数的原理。

      2. 数据类型注意：
        题目中 $a_i$ 可达 $2 \times 10^9$，如果 $n = 10^6$，糖果总数可达 $2 \times 10^{15}$，这远远超过了 int 的范围（约 21 亿）。因此必须使用 long long。

      3. 前缀和的妙用：
        通过前缀和将“环形”的相互依赖关系转化为了独立的常数项，这是本题从“模拟/搜索”转变为“贪心/数学”的破局点。

      4. 复杂度分析：

        • 读取数据：$O(n)$
        • 计算 $C$ 数组：$O(n)$
        • 排序：$O(n \log n)$
        • 计算结果：$O(n)$
          总复杂度为 $O(n \log n)$，对于 $10^6$ 的数据量，可以在 1 秒内跑完。
