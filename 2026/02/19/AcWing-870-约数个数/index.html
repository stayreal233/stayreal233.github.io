  1. 数论经典题：乘积的约数个数（算术基本定理）

    这是一道非常经典的数论题目，主要考察的是 ** 算术基本定理（唯一分解定理）** 的应用。

    要解出这道题，我们不需要真的把所有数乘起来（因为积会非常巨大，远超 long long 的范围），而是通过分解质因数的方法来间接计算。

    一、核心数学原理

    1. 算术基本定理

    任何一个大于 1 的自然数 N，都可以唯一分解成有限个质因数的乘积：

    N=p1^α1×p2^α2×⋯×pk^αk

    其中 pi 是质数，αi 是对应的指数（即这个质数出现了几次）。

    2. 约数个数定理

    如果一个数 N 已经写成了上面的质因数分解形式，那么它的约数个数为：

    约数个数=(α1+1)×(α2+1)×⋯×(αk+1)

    直观理解：

    对于每一个质因数 pi，构造约数时可以选择取 0 个、1 个…… 直到 αi 个，共有 (αi+1) 种选法。

    根据乘法原理，总选法就是各指数加一后的乘积。

    3. 多个数乘积的情况

    如果我们要算 a1×a2×⋯×an 的约数个数：

    • 分别对每一个 ai 进行质因数分解
    • 把所有数中相同质因数的指数累加
    • 再用约数个数公式计算

    示例： 6×8

    • 6=2^1×3^1
    • 8=2^3
    • 乘积分解：2^(1+3)×3^1=2^4×3^1
    • 约数个数：(4+1)×(1+1)=10

    二、解题步骤

    1. 用一个哈希表（C++ 中用 std::unordered_map / std::map），key 存质数，value 存该质数的总指数。
    2. 遍历每个输入数 ai，对其质因数分解，更新哈希表。
    3. 遍历哈希表，按公式 (α+1) 累乘，对 1e9+7 取模。

  • 四、关键点解析（新手必读）

    1. 为什么只枚举到 n

    一个数 n 最多只有一个大于 n 的质因数（否则乘积会超过 n）。

    循环到 i≤n/i 即可，最后若 n>1，说明剩下的本身是质数。

    时间复杂度从 O(n) 降到 O(n)。

    2. unordered_map 的作用

    • 自动去重、自动累加
    • 所有数分解出的同一个质因数，都累加到同一个 key 上

    3. 溢出与取模

    • 乘法中间结果会超 int，必须用 long long
    • 每次乘完立刻 % MOD，是大数取模题标准写法

    4. 时间复杂度

    • 对 n 个数，每个数分解 O(ai)
    • 总复杂度：O(nai)
    • 数据范围内运行极快，可轻松通过
