题解

思路：
每一项与第一项的差一定是d的倍数

#include
#include

int gcd(int a, int b){
return b ? gcd(b, a % b) : a;
}

int main(){
int n;

for(int i = 0; i < n; i ++){
    scanf("%d", &a[i]);
}

int d = 0;
for(int i = 1; i < n; i ++) d = gcd(d, a[i] - a[0]);

if(!d) printf("%d\n", n);
else printf("%d\n", (a[n - 1] - a[0]) / d + 1);

return 0;

}

辗转相除法求最大公约数

什么是最大公约数
最大公约数（Greatest Common Divisor）指两个或多个整数共有约数中最大的一个。也称最大公因数、最大公因子，a， b的最大公约数记为（a，b），同样的，a，b，c的最大 公约数记为（a，b，c），多个 整数的最大公约数也有同样的记号。求最大公约数有多种 方法，常见的有 质因数分解法、 短除法、 辗转相除法、 更相减损法。

辗转相减法求最大公约数
用(a，b)表示a和b的最大公因数：有结论(a，b)=(a，ka+b)，其中a、b、k都为自然数。

也就是说，两个数的最大公约数，将其中一个数加到另一个数上，得到的新数，其公约数不变，比如(4，6)=(4+6，6)=(4，6+2×4)=2.

要证明这个原理很容易：如果p是a和ka+b的公约数，p整除a，也能整除ka+b.那么就必定要整除b，所以p又是a和b的公约数，从而证明他们的最大公约数也是相等的.

基于上面的原理，就能实现我们的迭代相减法：(78，14)=(64，14)=(50，14)=(36，14)=(22，14)=(8，14)=(8，6)=(2，6)=(2，4)=(2，2)=(0，2)=2

基本上思路就是大数减去小数，一直减到能算出来为止，在作为练习的时候，往往进行到某一步就已经可以看出得值.

辗转相减到辗转相除
迭代相减法简单，不过步数比较多，实际上我们可以看到，在上面的过程中，由(78，14)到(8，14)完全可以一步到位，因为(78，14)=(14×5+8，14)=(8，14)，由此就诞生出我们的辗转相除法.

即：(a， b) = (a % b， b) = （b, a %b）

相当于每一步都把数字进行缩小，等式右边就是每一步对应的缩小结果。

对（a， b）连续使用辗转相除，直到小括号内右边数字为0，小括号内左边的数就是两数最大公约数。

代码
#include
#include
using namespace std;
int main()
{
int T;
cin >> T;
while(T–)
{
int a, b;
cin >> a >> b;
//辗转相除，直到小括号内右边数为0
while(b)
{
//c 一定小于 b
int c = a % b;
//小括号左边放除数，右边放约数
a = b;
b = c;
}
//小括号内左边数为最大公约数
cout << a << endl;
}

AcWing 871 约数之和

数论经典题：乘积的约数之和（约数之和定理）

这道题是上一题 “约数个数” 的延伸，核心考点依然是算术基本定理（唯一分解定理），重点考查约数之和定理的应用。

一、核心数学原理

1. 唯一分解定理

任何一个大于 1 的自然数 N，都可以唯一分解为：

N=p1^α1×p2^α2×⋯×pk^αk

其中 pi 是质数，αi 是该质数的指数（出现次数）。

2. 约数之和定理

若 N 的质因数分解式如上，则 N 的所有约数之和 S 为：

S=(p1^0+p1^1+⋯+p1^α1)×(p2^0+p2^1+⋯+p2^α2)×⋯×(pk^0+pk^1+⋯+pk^αk)

直观理解：

从每个括号中各选一项相乘，得到的就是 N 的一个约数；将所有组合的结果相加（即括号展开），就是所有约数的总和。

3. 多个数乘积的情况

对于 n 个数的乘积，只需先统计所有数中每个质因数的总指数，再套用约数之和定理即可 —— 无需直接计算大数乘积，避免溢出。

二、算法实现细节

公式中每一项 (p0+p1+⋯+pα) 无需暴力计算幂次，可通过如下简洁循环高效求解：

技巧原理（递推推导）：

  • α=0 时：t=1（对应 p0）
  • α=1 时：t=1×p+1=p+1（对应 p1+p0）
  • α=2 时：t=(p+1)×p+1=p2+p+1（对应 p2+p1+p0）
  • 以此类推，既简洁又避免大数幂运算的溢出风险。

三、C++ 代码实现

四、关键点解析

1. 指数与底数的双重关注

与 “约数个数” 仅需用到指数（α+1）不同，约数之和需要同时用到底数（质数 *p*）** 和指数（*α*）**，需注意两者的类型处理（均需转为 long long 避免溢出）。

2. 取模运算的陷阱

计算 t = (t * p + 1) % MOD 时：

  • p 虽是 int，但 t * p 可能超过 2×109（如 p 接近 2×109 时）；
  • 必须将 t 定义为 long long，或对乘法项强制类型转换，确保运算不溢出。

3. 计算顺序的合理性

先完整统计所有数的质因数及总指数，再统一计算约数之和 —— 因为约数之和定理针对的是 “最终乘积” 的整体质因数分解式，而非单个数字。

4. 时间复杂度分析

  • 质因数分解：n 次操作，每次 O(ai)；
  • 哈希表遍历：质因数个数远小于 n×log(ai)；
  • 总复杂度 O(nai)，在 、 时约为 4.5×106，完全满足 1 秒限时。

总结比较（与约数个数题对比）

题型 核心公式 核心差异
约数个数 ∏(αi+1) 仅需指数 + 1 后累乘
约数之和 ∏(∑j=0αipij) 需计算幂和后累乘

两者代码框架几乎一致，核心区别仅在于最后累乘的逻辑，均基于质因数分解的核心思想。

总结

  1. 约数之和定理的核心是 “质因数幂和的乘积”，需先统计所有数的质因数总指数；
  2. 用递推公式 t = (t * p + 1) % MOD 计算幂和，避免暴力幂运算和大数溢出；
  3. 所有乘法运算需用 long long 存储中间结果，并及时取模（109+7）。

  1. 数论经典题：乘积的约数个数（算术基本定理）

    这是一道非常经典的数论题目，主要考察的是 ** 算术基本定理（唯一分解定理）** 的应用。

    要解出这道题，我们不需要真的把所有数乘起来（因为积会非常巨大，远超 long long 的范围），而是通过分解质因数的方法来间接计算。

    一、核心数学原理

    1. 算术基本定理

    任何一个大于 1 的自然数 N，都可以唯一分解成有限个质因数的乘积：

    N=p1^α1×p2^α2×⋯×pk^αk

    其中 pi 是质数，αi 是对应的指数（即这个质数出现了几次）。

    2. 约数个数定理

    如果一个数 N 已经写成了上面的质因数分解形式，那么它的约数个数为：

    约数个数=(α1+1)×(α2+1)×⋯×(αk+1)

    直观理解：

    对于每一个质因数 pi，构造约数时可以选择取 0 个、1 个…… 直到 αi 个，共有 (αi+1) 种选法。

    根据乘法原理，总选法就是各指数加一后的乘积。

    3. 多个数乘积的情况

    如果我们要算 a1×a2×⋯×an 的约数个数：

    • 分别对每一个 ai 进行质因数分解
    • 把所有数中相同质因数的指数累加
    • 再用约数个数公式计算

    示例： 6×8

    • 6=2^1×3^1
    • 8=2^3
    • 乘积分解：2^(1+3)×3^1=2^4×3^1
    • 约数个数：(4+1)×(1+1)=10

    二、解题步骤

    1. 用一个哈希表（C++ 中用 std::unordered_map / std::map），key 存质数，value 存该质数的总指数。
    2. 遍历每个输入数 ai，对其质因数分解，更新哈希表。
    3. 遍历哈希表，按公式 (α+1) 累乘，对 1e9+7 取模。

三、 C++ 代码实现

  • 四、关键点解析（新手必读）

    1. 为什么只枚举到 n

    一个数 n 最多只有一个大于 n 的质因数（否则乘积会超过 n）。

    循环到 i≤n/i 即可，最后若 n>1，说明剩下的本身是质数。

    时间复杂度从 O(n) 降到 O(n)。

    2. unordered_map 的作用

    • 自动去重、自动累加
    • 所有数分解出的同一个质因数，都累加到同一个 key 上

    3. 溢出与取模

    • 乘法中间结果会超 int，必须用 long long
    • 每次乘完立刻 % MOD，是大数取模题标准写法

    4. 时间复杂度

    • 对 n 个数，每个数分解 O(ai)
    • 总复杂度：O(nai)
    • 数据范围内运行极快，可轻松通过

整体思路

该题的题解是整理了一下 y总视频里面讲解思路,希望对看完视频还不大理解的同学有点帮助
首先我们知道 如果 k == n ,那么就证明所有的数字是全部都选,
如果 k < n , 那么就要思考怎样去选择了:
1.k 如果是偶数的话,选出来的结果一定是非负数 , 原因如下:
(1) # 负数的个数是偶数个的话,负负得正,那么一定是非负数
(2) # 负数的个数如果是奇数个的话,那么我们就只选偶数个绝对值最大的负数
2.k 如果是奇数个的话,
(1)# 所有的数字如果都是负数,那么选出来的结果也一定都是负数
(2)# 否则的话,则一定至少有 1个非负数, 那么我们将最大的数取出来, 此时要选的个数就是 k–,
# k– 是偶数,那么就又转化为 k– 是偶数的情况思考

C++ 代码
#include
#include
#include
using namespace std;

typedef long long LL ;
const int N = 100010 , mod = 1000000009 ;
int a[N];

int main()
{
int n , k ;
scanf(“%d%d”,&n,&k);
for(int i = 0 ; i < n ; i ++) scanf(“%d”,&a[i]);

sort(a,a + n);

LL res = 1 ; //乘积初始化
int l = 0 , r = n - 1 ;//双指针初始化
int sign = 1 ; // 符号初始化

//由于4种情况除了 k 是奇数且 k < n 的时候需要特判一下处理一下符号 ,其他的时候都可以转化为双指针做法
//k 是奇数是先选出最大的数, k-- 就是偶数,两边再同时取对,转化成相同的双指针做法
if(k % 2 )
{
    res =  a[r]; // 取出最大的一个数
    r -- ; //右指针移动
    k -- ; //个数减1

    if(res < 0) sign = -1;  // 如果最大值都是负数,就证明全是负数,那么符号要发生改变
}
while(k) // 双指针做法
{
    LL x = (LL)a[l] * a[l + 1] , y = (LL)a[r] * a[r - 1];//两边同时取对
    //选择更大的一对,和归并排序思路相近
    if(x * sign > y * sign)
    {
        res = x % mod * res % mod; // 需要注意的是 :不可以写成(x * res) % mod ,也不可以写成是 res % mod * x % mod
                                   // 因为x最大是 10^10，如果不先取模的话，和res相乘的结果最大是 10^19,会暴long long。            
        l += 2; // 指针移动                                 
    }
    else 
    {
        res = y % mod * res % mod; 
        r -= 2; 
    }
    k -= 2; 
}
printf("%lld",res);
return 0;

}

灵能传输

在游戏《星际争霸 II》中，高阶圣堂武士作为星灵的重要 AOE 单位，在游戏的中后期发挥着重要的作用，其技能”灵能风暴“可以消耗大量的灵能对一片区域内的敌军造成毁灭性的伤害。

经常用于对抗人类的生化部队和虫族的刺蛇飞龙等低血量单位。

你控制着 n 名高阶圣堂武士，方便起见标为 1,2,⋅⋅⋅,n。

每名高阶圣堂武士需要一定的灵能来战斗，每个人有一个灵能值 ai 表示其拥有的灵能的多少（ai 非负表示这名高阶圣堂武士比在最佳状态下多余了 ai 点灵能，ai 为负则表示这名高阶圣堂武士还需要 −ai 点灵能才能到达最佳战斗状态）。

现在系统赋予了你的高阶圣堂武士一个能力，传递灵能，每次你可以选择一个 i∈[2,n−1]，若 ai≥0 则其两旁的高阶圣堂武士，也就是 i−1、i+1 这两名高阶圣堂武士会从 i 这名高阶圣堂武士这里各抽取 ai 点灵能；若 ai<0 则其两旁的高阶圣堂武士，也就是 i−1,i+1 这两名高阶圣堂武士会给 i 这名高阶圣堂武士 −ai 点灵能。

形式化来讲就是 ai−1+=ai,ai+1+=ai,ai−=2ai。

灵能是非常高效的作战工具，同时也非常危险且不稳定，一位高阶圣堂武士拥有的灵能过多或者过少都不好，定义一组高阶圣堂武士的不稳定度为 maxni=1|ai|，请你通过不限次数的传递灵能操作使得你控制的这一组高阶圣堂武士的不稳定度最小。

输入格式

本题包含多组询问。输入的第一行包含一个正整数 T 表示询问组数。

接下来依次输入每一组询问。

每组询问的第一行包含一个正整数 n，表示高阶圣堂武士的数量。

接下来一行包含 n 个数 a1,a2,⋅⋅⋅,an。

输出格式

输出 T 行。

每行一个整数依次表示每组询问的答案。
